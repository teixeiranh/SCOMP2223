========================================= readme.txt =========================================

Leia atentamente este ficheiro antes de iniciar o seu trabalho nas aulas PL de SCOMP.

Regras:
=======
a) Cada grupo deve ter o seu repositório.

b) O nome do repositório no Bitbucket segue o seguinte padrão: scomp2223tttgxx
	- ttt deve ser substituído pelo nome da turma. Ex: 2na
	- xx deve ser substituído pelo número do grupo (sempre com 2 dígitos). Ex: 01
	- Exemplo: scomp2223nag01

c) Estrutura de diretórios na raiz do repositório:
	- A estrutura de pastas inicial já está criada, uma pasta para cada ficha PL
	- Deve manter a estrutura e nomes das pastas que recebeu
	- Dentro da pasta de cada ficha DEVERÁ CRIAR a(s) subpasta(s) necessárias para cada exercício
	- O nome da pasta de cada exercício deve ser iniciado pelas letras MINÚSCULAS "ex" e ser seguido do número
	  (SEMPRE com 2 dígitos) que identifica o exercício. Exemplo: "ex01"

d) Durante a resolução das fichas deve preencher o ficheiro "problemas.txt" relativo à ficha em questão
	O ficheiro "problemas.txt" deverá referir quais os exercícios que não foram
	resolvidos e quais os problemas detetados na tentativa de resolução.

ATENÇÃO: Deve cumprir com rigor as regras enumeradas neste documento.
